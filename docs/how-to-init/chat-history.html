● The short answer: /init itself cannot be invoked programmatically, but its outcome can be replicated.                 
                                                                                                                        
  Here's what the docs say:                                                                                             
                                                                                                                        
  Why /init is interactive-only:                                                                                        
  - Slash commands (built-in and user-invoked) are only available in interactive mode                                   
  - The -p/--print non-interactive flag explicitly does not support slash commands                                      
                                                                                                                        
  What you can do programmatically:                                                                                     
                                                                                                                        
  # Replicate /init behavior via -p (non-interactive)                                                                   
  claude -p "Analyze this project and generate a CLAUDE.md file..." \                                                   
    --allowedTools "Read,Glob,Grep"                                                                                     
                                                                                                                        
  There is a --init-only flag, but that runs initialization hooks (pre-session shell commands from config), not the     
  CLAUDE.md generation — different thing entirely.                                                                      
                                                                                                                        
  So your approach is correct and is actually the recommended pattern. The docs explicitly say: "In -p mode, describe   
  the task you want to accomplish instead [of using slash commands]."                                                   
                                                                                                                        
  Your document-agent using a crafted prompt + claude -p is the proper way to automate this. The only downside vs /init 
  is that /init has hardcoded awareness of what makes a good CLAUDE.md — your CLAUDE-prompt.md compensates for that by  
  being explicit about what sections to include and what to look for in the codebase.
